# Contributing to conduit-vscode

Thanks for your interest in contributing. Follow these steps to get your changes considered:

1. Fork the repository and create a feature branch from `main`.
2. Run tests locally: `npm ci && npm test` and ensure all tests pass.
3. Follow the coding style in the repository. Keep changes focused and small.
4. Update docs/CHANGELOG.md with a short entry for your change.
5. Open a Pull Request against `main`. Include a clear description and link to any relevant issue.

PR checklist:
- [ ] Tests added/updated and passing
- [ ] Documentation updated (README, CHANGELOG) if behaviour changes
- [ ] Version bumped if releasing a new version
- [ ] No secrets in commits

Maintainers will review and request changes if necessary. For major changes, open an issue first to discuss design and scope.
